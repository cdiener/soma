General

Soma is built in a modular fashion. In order to control the included modules simply change the all.* file and recompile by running make in the less or sass (coming soon) folder. The files colors.* and general.* should always be included.
Soma applies some styles by default. This includes styling for links and fonts. Most of the styles in soma make use of four basic colors: gray, blue, green, red, which can be chnaged by modifying colors.less.By default Soma uses the systems sans-serif font, but you can easily use whatever font you like by including the following style in your head. For instance using Google Fonts you would use the following:

<head>
<link rel=stylesheet href=soma.css/>
<link href=//fonts.googleapis.com/css?family=Open+Sans|Inconsolata rel=stylesheet>
...
<style>
	body, textarea, input, select {
	font: 15px Open Sans, sans-serif;
	}
</style>

Please note that it is really hard to debug CSS against all possible fonts. If you find a Font combination where it looks odd please file a bug report.

The combination shown here works well and is fully supported.

Buttons (buttons.*)

Buttons are generated by applying the "btn" class followed by additional specifiers such as color and and smooth animation. Add the "sm" class for smaller buttons.

Code

<a href="#" class="btn blue smooth">Button blue</a>
<a href="#" class="btn green smooth">Button green</a>
<a href="#" class="btn red smooth">Button red</a>
<a href="#" class="btn smooth sm">Small button</a>

Navigation (nav.*)

Soma is designed for top navigation bars. Those bars are responsive by default and should look good on mobile and desktop. Additionally, Soma allows you to choose between two different toggle modes. A pure CSS implementation with a close button and Bootstrap-like mode where the navigation is opened and closed by clicking on the hamburger icon.

Pure CSS

<div class=nav tabindex=-1 onclick=this.focus()>
<div class="container">
	<a href="/" class="pagename">soma.css</a>
	<a href="index.html" class="smooth">Home</a>
	<a href="docs.html" class="current smooth">Docs</a>
</div></div>

Bootstrap-like

<div class=nav id=topnav tabindex=-1 onclick=toggle>
<div class="container">
	<a href="/" class="pagename">soma.css</a>
	<a href="index.html" class="smooth">Home</a>
	<a href="docs.html" class="current smooth">Docs</a>
</div></div>
...

Grid layout (grid.*)

Soma uses the same grid layout as min.css. The design principle is mobile-first with a screen divided into rows and 12 columns.

Code

<div class="row">    
	<div class="col c12">12</div>
</div>
<div class="row">
	<div class="col c4">4</div><div class="col c8">8</div>
</div>

Timelines (timeline.*)

Timelines are used to structure temporally ordered data in a clear fashion.

Code

<ol class="timeline">

Cards (cards.*)

Cards give structure to your elements and allow you to separate page elements visually.

Code

<div class=card>
<h1>Header</h1>
<h2>Subheader</h2>
<h3>Colored subheader</h3>
<p>And some text...</p>
</div>

<div class="card neutral">
<h1>Neutral colored</h1>
<h2>Still the same</h2>
<h3>Colored subheader</h3>
<p>And some neutral statement...</p>
</div>

<div class="card pos">
<h1>Positive message</h1>

</div>

<div class="card neg">

Icons (icons.*)

Soma does not come with a complete icon solution since I feel that this depends a lot on your project. If you need a lot of icons a public icon font such as FontAwesome might be a good solution. Its size is significant but due to its ubiquitious use there is a good chance your site visitor has it cached. If you only want some particular icons you might want to generate your own icon font with icomoon. Soma includes a small icon font as optional module. In order to use it add icons.less to all.less and include the font folder in your project.

Code

<i class="icon i-linkedin"></i>
<i class="icon i-mail"></i>
<i class="icon i-reddit"></i>
<i class="icon i-credit-card"></i>
